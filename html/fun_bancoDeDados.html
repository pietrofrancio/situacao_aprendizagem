Introdução

Os bancos de dados estão no coração de praticamente todos os sistemas digitais modernos — aplicativos, sites, empresas, escolas, redes sociais e até jogos dependem deles para organizar, armazenar e recuperar informações de forma rápida e segura.

Aqui, você vai entender como os dados são representados, como se relacionam entre si e como são manipulados por meio de linguagens específicas, como o SQL.

A proposta é desenvolver uma compreensão sólida e prática sobre como bancos de dados funcionam na vida real.

    • Modelagem de Banco de Dados

    A modelagem de banco de dados é o processo responsável por estruturar e organizar as informações que serão armazenadas em um sistema. Ela define como os dados serão representados, quais entidades compõem o negócio, quais atributos cada uma possui e quais relações existem entre elas.

    Esse planejamento é fundamental para garantir que o banco de dados seja eficiente, consistente, seguro e capaz de atender às necessidades reais do projeto. Por meio da modelagem, é possível evitar redundâncias, reduzir erros, otimizar consultas e assegurar que o sistema cresça de forma organizada.

    Ao estruturar corretamente os dados desde o início, a modelagem se torna uma etapa essencial para o desenvolvimento de soluções tecnológicas robustas, escaláveis e alinhadas aos objetivos da aplicação.

    Modelagem DER

    A modelagem DER (Diagrama Entidade-Relacionamento) é uma forma visual de representar como os dados de um sistema se organizam e se conectam. Ela mostra:

  • Entidades (como tabelas: Cliente, Produto, Personagem)
  • Atributos (informações dessas entidades)
  • Relacionamentos (como uma entidade se liga à outra, ex.: 1:N, N:N)

    • Explicação do Diagrama DER

    O diagrama apresenta um sistema que organiza personagens, suas editoras, criadores, cidades e grupos. Nele, existem alguns elementos:

  • Personagem: Entidade central. Seus atributos são código, nome de herói, alter ego, apelido, habilidade, arma.
  • Editora: Cada personagem pertence a uma única editora, enquanto uma editora pode ter vários personagens.
  • Cidade: Todo personagem está ligado a uma cidade, e uma cidade pode conter muitos personagens.
  • Criador: Relacionamento N:N, onde personagem pode ter vários criadores e um criador pode ter criado vários personagens. A relação registra o ano de criação.
  • Grupo / Equipe: Também possui um relacionamento N:N.

    • Modelagem MER

    A modelagem MER (Modelo Entidade-Relacionamento) é a etapa em que representamos, de forma visual e organizada, as entidades de um sistema, seus atributos e como elas se relacionam entre si.

  • MER = é o modelo conceitual, a descrição lógica das entidades e relacionamentos.
  • DER = é o desenho gráfico desse modelo, mostrando visualmente esses elementos (retângulos, linhas, cardinalidades etc.).

    • Explicação do Diagrama MER

    Imagine um sistema de vendas. Antes de criar qualquer banco de dados, usamos a modelagem MER para identificar quais informações serão armazenadas e como elas se relacionam.

  • Entidades: São os elementos principais do sistema (Cliente, Pedido, Produto).
  • Atributos: São as informações que pertencem a cada entidade (Cliente: ID_Cliente, Nome, Email; Pedido: ID_Pedido, Data, ValorTotal; Produto: ID_Produto, NomeProduto, Preço).

    • Relacionamentos: Mostram como as entidades se conectam:

  • Cliente faz Pedido → relação 1:N (um cliente pode fazer vários pedidos)
  • Pedido contém Produto → relação N:N (um pedido pode ter vários produtos e um produto pode aparecer em vários pedidos)
